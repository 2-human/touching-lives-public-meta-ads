# review-widget composition manifest — public/meta-ads/

This is a **cross-project adoption**, not a fresh library composition.
Glinda has been through more iteration than touching-lives' current
library, so we brought over Glinda's *composed* widget files verbatim
rather than re-composing from scratch. Two library atomics were synced
alongside the composed files so a future re-composition from
touching-lives' own library would land on the same shape.

## Adjustments applied during the adoption

### Config-global rename

Glinda's config is exposed on `window.GLINDA_CONTACT_CONFIG`. Touching
Lives' is `window.TOUCHING_LIVES_CONTACT_CONFIG`. Every
`GLINDA_CONTACT_CONFIG` reference in the three widget JS files was
renamed:

- `review-bootstrap.js` — `window.GLINDA_CONTACT_CONFIG` reads + the
  `hintFor()` error string + the `configGlobalName` argument passed to
  `init()`.
- `review-mode.js` — no direct references (the config global name is
  passed in via the `configGlobalName` init arg).
- `contact-form.config.js` — the assignment target.

### Brand tokens

Kept Glinda's purple / indigo palette verbatim in `review-mode.css`
(`--review-primary-deep: #8705E4`, banner gradient
`#8705E4 → #6804B5`, etc.). The `review-bootstrap.js` inline entry-button
`<style>` block was also left with Glinda's purple. Visual differentiation
from Touching Lives' brand navy is acceptable for this first cut, and
matching Glinda pixel-for-pixel is what the adoption asked for. Re-toning
to touching-lives navy is a future pass, not this task.

### Firebase config

Preserved the touching-lives Firebase project values from the prior
`contact-form.config.js` (`touching-lives-84345`, apiKey
`AIzaSyBwf796wzoUojSh7uBmqa5leBWIFZo58vs`, databaseURL
`https://touching-lives-84345-default-rtdb.firebaseio.com`, etc.).
Glinda's Firebase project (`glinda-website`) was NOT copied over.

### Labels

Copied Glinda's `REVIEW_LABELS` shape (canonical keys — `bannerText`,
`sidebarTitle`, `modalCommentPlaceholder`, `toggleButton`, etc.) rather
than the prior touching-lives snake_case shape (`banner_title`,
`empty_state_pending`, etc.), because the composed widget in
`review-mode.js` reads the Glinda-canonical keys. Glinda-specific
strings ("Reviewing the Glinda Method", "Kindness is Cooler than
Coolness.") were rewritten for the ad-review context or dropped in
favour of neutral defaults:

| Glinda string | Touching-lives string |
| --- | --- |
| "Reviewing the Glinda Method" | "Ad review mode" |
| "Kindness is Cooler than Coolness." | (dropped — no house signature) |
| "No comments yet. Click any paragraph to start." | "No comments yet. Hover any element to add one." |

Generic strings ("Save comment", "Cancel", "Archived", "Delete",
"Comment updated.", "Comments") kept verbatim. Where a suitable string
already exists in `DEFAULT_LABELS` inside `review-mode.js`, it was
omitted from the config to fall through to the default.

### Behaviour keys

Set per the adoption instructions:

- `commentLifecycleMode: 'feedback-only'`
- `commentableContent: 'direct-text'`
- `ANCHOR_TAGS_EXTRA: ['label', 'blockquote', 'cite', 'figcaption', 'button']` (5 entries)
- `chromeAnchored: false`

`chromeAnchored` differs from Glinda's `true` because ad-preview pages
don't have real shared site chrome; keeping it off means chrome comments
wouldn't cross-page-surface anyway.

## Glinda variations inherited as-is

Glinda's composed output diverges from the canonical library spec in
several places (see `glinda/public/method/.composition-manifest.md` for
Glinda's own history). Those variations come across into touching-lives
untouched by this adoption:

- Banner exit affordance is `<a href="#exit">` not `<button>` (manual
  close via link).
- Modal has no header close button — Escape / backdrop click / Cancel
  button close it.
- Single center-bottom toast (not a bottom-right stack), 2.6s
  auto-dismiss.
- Button geometry: 6px border-radius, 9px 18px padding, font-weight 500.
- Spotlight pulse duration 1.5s (glinda's `spotlight-pulse.md` override
  vs library default 4s).
- Purple / coral / cyan / indigo brand palette (see `Brand tokens`
  above).
- Sidebar reserves 360px right gutter via `body { padding-right }` (not
  overlay).
- Pill is a 22px circle with `+` glyph.

## Behaviour recap

Runtime behaviour resulting from the config keys above:

- Any element with direct text of ≥ 2 chars gets a pill on hover — no
  need to opt in by tag.
- Site chrome (nav, header[role="banner"], footer) is skipped.
- Sidebar buttons on pending comments: Edit + Delete (no Apply /
  Archive; those transitions happen via the operator working the RTDB).
- Group-footer bulk-archive button is suppressed.
- Comments persist in Firebase RTDB at
  `touching-lives-84345/comments/{pushId}`.
- Anchor IDs shaped `{pageSlug}-{tag}-{n}` for content, `chrome-{tag}-{n}`
  for chrome (unused here since `chromeAnchored: false`).

## Re-adoption triggers

Re-run this adoption when:

- Glinda's `public/method/review-{bootstrap,mode.css,mode.js}` change
  meaningfully upstream (glinda re-composition, new atomic landing,
  brand-token flip). Diff glinda → touching-lives, port the delta, keep

  glinda's — at that point flip from cross-project adoption to fresh
  library composition and retire this manifest's `adoption_type:
  cross-project`.
