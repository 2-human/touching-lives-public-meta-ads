## Deviations from canonical (RESOLVED — both landed upstream 2026-07-02)

Both project-specific deviations this composition surfaced were promoted to the canonical library on 2026-07-02 (commit `c818cac`) and synced back down into this repo on 2026-07-02 (project-side [test-session], mirroring the uzb hygiene pass). They are no longer deviations — the canonical library now documents them.

1. **Base-relative widget-asset resolution (hub/detail wiring).** `review-bootstrap.js` resolves `review-mode.css` / `review-mode.js` relative to **its own `<script src>`**, not the current page, because TL detail pages live at `/details/{slug}.html` while the widget files sit at the mirror root — a bare `review-mode.css` would 404 from a detail page. credo's pages sit next to the widget so it never hit this. **Landed upstream as G5** (`FEATURE.md` §"What composition produces" + `inert-entry-button.md`: the bootstrap MUST resolve assets relative to its own `<script src>` when pages sit at a different directory depth than the widget files). **Verified live 2026-07-02** — see Parity result below.
2. **Chrome-root name `.review-floating-pill` / `.review-toggle-btn`** for the entry pill (vs the `.review-pill` example in `css-isolation.md`). Both keep the `.review-` chrome-root prefix, so the isolation contract holds; only the example name differs. **Landed upstream as G5b** (`css-isolation.md`: the `.review-*` example class names are illustrative, not mandated; `.review-floating-pill` is named as a conformant instance).

## Wiring

Every generated page (`feed-preview.html` at hub, `details/{slug}.html` per ad) loads two scripts in `<head>`:

```html
<script src="{rel_prefix}review.config.js" defer></script>
<script src="{rel_prefix}review-bootstrap.js" defer></script>
```

Emitted by `work-setup/build-feed-preview.py :: review_widget_scripts(rel_prefix)` — `rel_prefix` empty for the hub, `../` for detail pages. The variant switcher (Image/Video + ratio) on detail pages keeps each variant zone's `data-comment-id` in sync and sets `data-variant-note` (e.g. `"Photo · 9:16"`); `__rwReveal` flips the switcher so a sidebar comment reveals the right variant with dual-highlight, per `variant-anchors.md` / `click-to-reveal.md`.
